Turbo Whisper 当前设计说明

只记录现在的软件是什么样，不讨论未来目标。

Current design snapshot · v1.0.x

一句话理解

这是一个常驻后台的语音输入工具。

当前 Turbo Whisper 的核心体验是：用户在任何应用里按全局热键，说话，应用录音并调用 Whisper API， 然后把转写出来的文字复制到剪贴板，并可自动输入到原本聚焦的窗口。

现在包含哪些部分

这个仓库当前主要由一个 PyQt6 桌面应用、录音模块、Whisper API 客户端、热键监听和自动输入模块组成。

运行形态悬浮窗 + 系统托盘，常驻后台。
触发方式全局热键开始或停止录音。
识别方式调用 OpenAI 兼容 Whisper 接口。
输出方式剪贴板和自动输入当前窗口。

主要操作流程

现在的设计围绕一条很短的链路展开。

1
启动应用应用创建 Qt 窗口和系统托盘，启动全局热键监听；如果启用 Docker autostart，会异步启动本地 Whisper 容器。
2
按热键录音热键触发后，悬浮窗口显示，PyAudio 开始从麦克风读取声音。
3
停止并转写再次触发或结束录音后，音频被封装成 WAV，发送到 Whisper API。
4
输出文本转写结果进入历史记录；按配置复制到剪贴板，并自动输入到当前窗口。
5
处理 Claude Code如果启用 Claude 集成，应用会等待本地 ready 信号，避免在 Claude 忙时直接输入。

当前设置项

这些是现在代码里已经存在或配置文件里支持的设置。

APIapi_url 和 api_key，支持 OpenAI 或自托管 faster-whisper-server。
Dockerdocker_autostart、容器名和 docker_run_command，用于可选管理本地 Whisper 服务。
热键hotkey，非 Windows 默认 Alt+Space，Windows 默认 F8。
音频采样率、声道、chunk size、输入设备、麦克风增益。
界面窗口宽高、背景颜色、波形颜色、录音时是否显示窗口。
行为是否自动输入、是否复制到剪贴板、语言、打字延迟。
历史和集成最近 20 条转写、录音保存、Claude 集成端口和等待超时。

当前入口和菜单

用户主要通过悬浮窗和托盘菜单控制应用。

系统托盘显示应用状态和热键提示，提供后台常驻入口。
Show Window手动显示悬浮窗口。
Start Recording从托盘菜单开始或停止录音。
Settings...打开窗口并展开设置面板。
Quit停止热键、录音资源、集成服务；如果本次会话启动过 Docker 容器，也会按配置停止它。

代码文件对应关系

如果只是想理解现在的设计，先看这些文件就够了。

main.py应用入口、悬浮窗口、设置面板、托盘菜单、录音到转写的主流程。
config.py配置默认值、配置文件路径、历史记录和录音文件保存。
recorder.py麦克风设备、PyAudio 录音、声音级别和 WAV 输出。
api.py把音频上传到 Whisper API，并把返回 JSON 里的 text 取出来。
docker_service.py可选 Docker 生命周期管理：启动、复用、等待端口就绪和退出时停止。
hotkey.py跨平台全局热键监听。
typer.py / integration_server.py自动输入、剪贴板，以及 Claude Code ready/busy 本地 HTTP 信号。

这个页面只描述当前设计。未来如果要设计目标状态，建议另开一个更轻量的草稿，而不要把目标编辑器塞进这里。
