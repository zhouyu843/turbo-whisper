= Turbo Whisper Troubleshooting Guide
:toc:

== Common Problems and Solutions

[cols="1,2"]
|===
|Problem |Solution

|**Microphone not picking up audio on Linux (PipeWire)**
|Install `pipewire-alsa` package: `sudo apt install pipewire-alsa`. This bridges PyAudio/ALSA to PipeWire. Restart PipeWire after: `systemctl --user restart pipewire wireplumber`

|**Microphone levels always zero**
|Check system audio settings. The mic might be muted or the wrong device selected. Use `wpctl status` to see available sources and `wpctl set-default <id>` to switch.

|**Auto-paste not working on KDE Wayland**
|Turbo Whisper uses evdev for Wayland typing. Set up permissions: `sudo usermod -aG input $USER`, create udev rule (see below), log out and back in.

|**evdev error: "Permission denied for /dev/uinput"**
|evdev needs write access to `/dev/uinput`. Run: `sudo usermod -aG input $USER` and create udev rule: `echo 'KERNEL=="uinput", GROUP="input", MODE="0660"' \| sudo tee /etc/udev/rules.d/99-uinput.rules` then `sudo udevadm control --reload-rules && sudo udevadm trigger`. Log out and back in.

|**evdev error: "cannot be opened for writing"**
|Same as above - `/dev/uinput` needs proper permissions. Check with `ls -la /dev/uinput` - should show `crw-rw---- root input`.

|**wtype error: "Compositor does not support virtual keyboard protocol"**
|KDE Plasma doesn't support wtype. Turbo Whisper uses evdev instead, which works on all Wayland compositors.

|**xdotool types into wrong window**
|xdotool only works for XWayland apps, not native Wayland apps like VS Code or Ghostty. Ensure evdev is working (see above) for native Wayland support.

|**Multiple app instances running**
|Kill all instances: `pkill -9 -f turbo_whisper`. Remove lock file: `rm -f $XDG_RUNTIME_DIR/turbo-whisper.lock`. Restart app.

|**Hotkey not working on Wayland**
|pynput works via XWayland on Wayland. Ensure XWayland is running. Check if hotkey conflicts with system shortcuts in KDE Settings > Shortcuts.

|**Transcription returns "You" or wrong text**
|The microphone isn't capturing audio properly. Check audio levels in the app UI. If levels are near zero, see "Microphone not picking up audio" solution.

|**Audio device indices keep changing**
|PipeWire can reassign device indices dynamically. Use the device name in the app's dropdown rather than relying on a specific index. Or set `input_device_index: null` in config to use system default.

|**App window doesn't appear when pressing hotkey**
|Check if another instance is running: `pgrep -af turbo`. Kill stale instances. Verify hotkey in config matches what you're pressing.

|**Transcription API errors**
|Verify API URL and key in settings. For self-hosted whisper, ensure the server is running. Check network connectivity.

|**Docker autostart says Docker CLI is not available**
|Install Docker Desktop or the Docker CLI package for your platform. Confirm `docker --version` works in the same environment used to launch Turbo Whisper.

|**Docker autostart fails because the daemon is not running**
|Start Docker Desktop or the Docker daemon, then relaunch Turbo Whisper. On Linux, check `systemctl status docker` or your rootless Docker service.

|**Docker container starts but API never becomes ready**
|Check that the configured `api_url` host and port match the Docker port mapping, for example `http://localhost:8000/v1/audio/transcriptions` with `-p 8000:8000`. The first model download can also exceed the default timeout; increase `docker_start_timeout_seconds` and watch `docker logs turbo-whisper-faster-whisper`.

|**Docker reports port 8000 is already in use**
|Another process or container is already bound to port 8000. Stop it with `docker ps` / `docker stop <name>`, or change both the Docker `-p host:8000` mapping and Turbo Whisper `api_url` to a free port.

|**First local transcription is very slow**
|faster-whisper-server may be downloading and loading the model for the first time. Use a persistent Hugging Face cache volume and allow a longer `docker_start_timeout_seconds` for first launch.

|**Slow recording start**
|First recording may be slow due to model loading on the API side. Subsequent recordings should be faster. Consider using a local whisper server for lower latency.

|**macOS: Auto-type not working**
|PyAutoGUI requires accessibility permissions. Go to System Preferences → Security & Privacy → Privacy → Accessibility, and add your terminal or the app to the allowed list.

|**macOS: Hotkey not detected**
|pynput requires Input Monitoring permissions on macOS. Go to System Preferences → Security & Privacy → Privacy → Input Monitoring and add your terminal or the app.

|**macOS: Text not inserted into original app after dictation**
|The recording orb can steal focus from the app you were typing in. Turbo Whisper saves the frontmost app when recording starts and re-activates it before pasting. Update to the latest version, restart the app, and ensure **Accessibility** is granted to your terminal or Turbo Whisper. If it still fails, set `show_window_on_recording` to `false` in `~/.config/turbo-whisper/config.json` (waveform hidden, focus stays put).

|**Windows: Auto-type not working**
|Ensure PyAutoGUI is installed. Some apps with elevated privileges may block simulated keyboard input. Try running Turbo Whisper as administrator.

|**PPA: "Repository does not have a Release file"**
|The PPA may not have packages for your Ubuntu version. Check which series are supported at https://launchpad.net/~bengweeks/+archive/ubuntu/turbo-whisper. Request support for your version or install from source.

|**PPA: "File already exists with different contents"**
|When uploading a new debian revision for the same upstream version, the orig.tar.gz must be identical. Download the existing one from Launchpad and use `-sd` flag with debuild to create a diff-only upload.

|**PPA: Package not appearing after upload**
|PPA builds take 5-15 minutes. Check build status at Launchpad. After build completes, wait 1-2 more minutes for repo sync before running `apt update`.

|**AUR: Package description not updating in search**
|AUR search indexes update periodically. The package page shows the new description immediately, but search results may take hours/days to reflect changes.

|**App not in start menu after install**
|The desktop file should be at `/usr/share/applications/turbo-whisper.desktop`. Run `update-desktop-database` or log out/in. Check with: `ls -la /usr/share/applications/turbo*`

|**App not starting on login**
|Copy desktop file to autostart: `cp /usr/share/applications/turbo-whisper.desktop ~/.config/autostart/`. Ensure the file contains `X-GNOME-Autostart-enabled=true`.

|**Window steals focus when recording starts (Linux)**
|Some Linux window managers don't fully respect Qt's `WindowDoesNotAcceptFocus` flag. The recording window may briefly take focus when appearing. The transcribed text should still be inserted into the correct window when recording stops, as focus is tracked internally.

|**Windows: Default hotkey conflicts**
|On Windows, the default hotkey is F8 (not Alt+Space). Alt+Space opens the window menu, Ctrl+Shift+Space conflicts with various apps, and Win+Shift+V conflicts with clipboard history. Change the hotkey in settings if F8 conflicts with other apps.

|**"Show Window" from tray doesn't bring window to front**
|On some Linux window managers, the window may not come to the foreground. Click "Settings" instead (which activates the window), or use the hotkey. You can also click directly on the window in your taskbar.

|**Terminal freezes when typing long text (Ghostty, etc.)**
|Some terminals may freeze when receiving rapid keyboard input via evdev. The default is 5ms per keystroke. Increase `typing_delay_ms` in your config (e.g., 10-20ms) if your terminal freezes. Alternatively, disable `auto_paste` and use Ctrl+V to paste from clipboard.
|===

== Diagnostic Commands

[source,bash]
----
# Check PipeWire status
wpctl status

# List audio input devices
pactl list sources short

# Test microphone with PipeWire directly
pw-record --target=@DEFAULT_SOURCE@ test.wav

# Test microphone with ALSA
arecord -d 3 test.wav

# Check if pipewire-alsa is installed
dpkg -l | grep pipewire-alsa

# Restart PipeWire
systemctl --user restart pipewire wireplumber

# Check running turbo-whisper processes
pgrep -af turbo

# View app logs
journalctl --user -f | grep -i turbo
----
