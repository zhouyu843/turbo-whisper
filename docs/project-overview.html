Turbo Whisper

一个跨平台的语音输入与转写桌面应用

Python · PyQt6 · Whisper API · Global Hotkey

这个项目到底是什么

把你说的话，变成当前窗口里的文字。

Turbo Whisper 不是一个传统网页应用，而是一个类似 SuperWhisper 的桌面语音听写工具。 你在任何软件里按下全局热键，它会弹出一个悬浮录音窗口，录下麦克风声音，发给 OpenAI 兼容的 Whisper 转写接口， 再把识别出的文本自动输入到你原本正在使用的窗口里。

输入方式 按热键说话，不需要切换应用。
识别核心 调用 OpenAI Whisper 或自托管 faster-whisper-server。
输出结果 自动打字到当前窗口，同时可复制到剪贴板。

核心流程

下面这五步就是整个应用的主线，代码基本也围绕这条链路组织。

1 监听热键 hotkey.py 用 pynput 或平台后端监听全局快捷键。
2 录制音频 recorder.py 通过 PyAudio 采集麦克风并计算音量。
3 显示悬浮窗 waveform.py 和 main.py 绘制声音响应动画。
4 调用 Whisper api.py 把 WAV 音频发送到 OpenAI 兼容接口。
5 输入文字 typer.py 将识别文本粘贴或模拟键盘输入。

代码模块怎么分工

如果你之后要读代码，可以先从这些文件入手。

main.py

应用入口，负责 Qt 应用、托盘菜单、悬浮窗口、设置面板和整体流程调度。

config.py

读写配置文件，保存 API 地址、热键、语言、历史记录和录音文件设置。

recorder.py

麦克风采集层，输出 WAV 字节，并把实时音量传给界面动画。

api.py

Whisper 客户端，使用 httpx 上传音频并拿回转写文本。

hotkey.py

全局快捷键层，处理 Linux、macOS、Windows 上的触发逻辑。

typer.py

输出层，把转写结果输入到当前焦点窗口，失败时退回剪贴板。

适合拿来做什么

  • 对着 Claude Code、Aider、终端或编辑器直接说需求。
  • 说完后文本会进入当前窗口，像你手动打出来一样。
  • 仓库里还有 Claude Code integration，用本地 HTTP 信号等待 Claude 空闲后再输入。
  • 减少长时间键盘输入，适合 RSI、腕管综合征或手部不适场景。
  • 全局热键加系统托盘，让它像后台工具一样一直待命。
  • 历史记录会保存最近转写内容，方便再次复制或复用。
  • Whisper 支持多语言识别，配置里可设置语言。
  • 可以使用 OpenAI 官方接口，也可以接自托管 faster-whisper-server。
  • 录音文件可跟随历史记录保存，便于回听和排查。

最关键的配置长这样

{
  "api_url": "https://api.openai.com/v1/audio/transcriptions",
  "api_key": "sk-your-api-key",
  "hotkey": ["alt", "space"],
  "language": "en",
  "auto_paste": true,
  "copy_to_clipboard": true,
  "waveform_color": "#84cc16"
}

技术栈一眼看懂

这个项目的复杂度主要来自“跨平台桌面能力”，不是来自业务逻辑。

PyQt6：窗口、托盘、设置面板
PyAudio：麦克风录音
httpx：调用转写 API
pynput / evdev / pyautogui：热键与输入

总结：Turbo Whisper 是一个“语音到文字再自动输入”的桌面工具。网页只是这份项目说明； 真正的应用运行入口是 turbo-whisper 或 python -m turbo_whisper.main。
